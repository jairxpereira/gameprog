13. O tamanho dos tipos de dados

Os diversos tipos básicos de dados ocupam uma quantidade  de  memória
maior ou menor conforme o alcance de  suas  faixas  de  valores; essa
quantidade de memória é dada no tamanho em bytes que  o  tipo  ocupa.

Em primeiro lugar o que determina a grandeza da faixa de valores  que
o tipo alcança é o  número  de  bits  ou  bytes  que  o  tipo  ocupa.

Sabemos que um byte é um conjunto de 8 bits; para saber  a  faixa  de
valores  que  um  byte  pode  alcançar  nós  realizamos  um  operação 
matemática de potenciação, tendo 2 como base e o número  de  bits  do
tipo como expoente.   Desta forma um byte é igual  a  28,  que dá 256,
valores  de  0  a  255.   O  unsigned char  tem  o tamanho de um byte.

Os subtipos signed's  têm o mesmo número de bits  que as contrapartes
unsigned's, porém um bit é reservado para indicar o  sinal do número,
se for zero é positivo, se for um 1 é negativo.  Com um bit a menos o
tipo perde metade de sua faixa  de  valores  para  abrigar os números
negativos; assim a faixa de valores obtida pelo signed  char  é  dado
por 27 que é igual a 128,  tendo  o primeiro bit  como  indicador  do
sinal do número.

O tamanho de uma estrutura é dado pela soma do tamanho dos  tipos  de
seus elementos individuais.

É importante saber o tamanho dos tipos para compreender  questões  de
gasto de memória:
O tipo int ocupa 4 bytes, assim para guardamos uma lista de mil int's
precisaremos de 4.000 bytes de memória.  Nessa lista sabemos  que  um
int é encontrado a cada quatro bytes.



Na linguagem  C  há  a  instrução  sizeof,  conhecida  como  operador
de tamanho, cuja sintaxe é igual ao de  uma  função,  que  retorna  o
tamanho  em  bytes  da  variável  apresentada.  Sua  sintaxe  geral é
assim:
 
 nTam = sizeof(vVar);
 nTam = sizeof(ncx);

 nTam.........Tamanho de vVar retornado na forma de  um  unsigned int.
 vVar.........Variável  de  qualquer  tipo,  inclusive  estrutura.

O uso  de  sizeof()  para obtermos o tamanho tipo é providencial pois
conforme  o compilador e o sistema operacional o tamanho do tipo pode
variar,  diferença  essa  que  pode  quebrar  nosso  programa.

/* Ilustra sizeof() - tamanho dos tipos */
#include "stdio.h"
#include "stdlib.h"
#include "string.h"

/* Protótipo de initSys() */
void initSys( char *title);

/* Estrutura Funcionario */
struct Funcionario {
char sNome[40];
char sEndereco[40];
float nSalario;
int nIdade;
} xVendedor;

int main(void)  {

/* Declaração de alguns tipos */
char nByte;
int nInteiro;
long nLongo;
float nFloat;
double nDouble;

initSys("title tamtypes.c");

/* Mostra o tamanho dos tipos mais usados */
printf(" Char...: \t %d byte\n",sizeof(nByte));
printf(" Inteiro: \t %d bytes \n",sizeof(nInteiro));
printf(" Longo..: \t %d bytes \n",sizeof(nLongo));
printf(" Float..: \t %d bytes \n",sizeof(nFloat));
printf(" Double.: \t %d bytes \n\n",sizeof(nDouble));
printf(" Estrutura Funcionario: %d bytes \n",sizeof(xVendedor));

getchar();
return 1;
}


/*************** Função initSys()  */
void initSys( char *title)
{
/* Configura a janela */
system(title);
system("color F1");
printf("\n\n");
}
