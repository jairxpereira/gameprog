05-1. Introdução à funções e métodos
1.1 Anatomia das funções e métodos


Funções e métodos são tratados como sinônimos neste curso.  Porém, saiba 
que métodos são funções dentro de classes e objetos. As variáveis dentro
de uma classe são chamadas atributos ou propriedades.  Vamos conservar o 
nome atributo porque no C#  as  propriedades  podem ser enriquecidas com 
código de verificação e validação de seus valores.

Juntos, métodos, atributos, propriedades e demais estruturas que possam
aparecer dentro uma classe são chamados de membros da classe. 

A anatomia das funções e métodos em  c#  é  semelhante  à  linguagem  c++.
Há os modificadores que definem a visibilidade e  o  nível  de  acesso  da
função, há o tipo de retorno, nome da função e os parâmetros ou argumentos
de entrada da função.	

1.2. Classes e Funções estáticas
static void Main(string[] args)

Em C# há classes que não geram objetos  e  que  podem  ser  usadas  sem  a 
necessidade de serem instanciadas em uma variável. Um bom exemplo de classe
assim é a classe Console que temos usado para realizar a saída de dados em
nossos programas. Para ter esse comportamento essa classe e seus métodos são
definidos como static

Resumindo, funções estáticas não são herdadas por objetos particulares e
podem ser utilizadas diretamente sem a necessidade da criação de um objeto.

É comum utilizar funções estáticas para tratar com informações ligadas a
classe em si. Por exemplo, você pode utilizar uma função estática para
controlar e retornar o número de objetos criados de uma classe.	O número
de objetos vai ser representado por uma variável inteira também estática.

Vamos ilustrar tudo isso com um exemplo. Vamos construir uma classe
Ferramentas que vai retornar o nome do usuário local e o nome de 
rede do computador sendo usado. Essa classe vai ter uma variável
inteira que vai contar quantas vezes suas utilidades são usadas.
Nesta classe vamos ter um método para configurar a janela de console.

Veja abaixo um simples esquema gráfico dessa classe:

Todos os métodos e propriedades dessas classes serão públicas e
estáticas.

Veja agora o código dessa classe:


  static class Ferramentas
  {

    static public int nvezes = 0;

    // Retorna o nome de rede do computador    
    public static String nome_computador()
    {   
      String nome_pc = "";
      nome_pc = Environment.MachineName;
      nvezes++;
      return nome_pc;    
    } // nome_computador().fim

    // Retorna o nome do usuário local
    public static String nome_usuario()
    {
    String usuario = "";
      usuario = Environment.UserName;
      nvezes++;
      return usuario;
    } // nome_usuario().fim

    // Método para configurar a janela
    public static void config_janela(String titulo)
    {
      // Vamos configurar a janela
      Console.ForegroundColor = ConsoleColor.Blue;
      Console.BackgroundColor = ConsoleColor.White;
      Console.Title = titulo;
      Console.SetBufferSize(80, 30);
      Console.SetWindowSize(80, 30);
      Console.Clear();
      nvezes++;
    } // config_janela() fim    

  } // fim da classe



Perceba que a classe com seus métodos e atributos podem ser utilizados
sem a necessidade de se declarar um objeto antes:


// projeto stsStatic - Arquivo: Program.cs
// Esse projeto ilustra como usar classes e métodos estáticos
using System;

namespace tstStatic
{
  class Program
  {
    
    static void Main(string[] args)
    {

      Ferramentas.config_janela("projeto tstStatic");
      
      Console.WriteLine("\n Nome do usuário: " + 
        Ferramentas.nome_usuario());

      Console.WriteLine(" Nome do computador: " +
        Ferramentas.nome_computador());

      Console.WriteLine(" Contagem de uso das ferramentas: " +
        Ferramentas.nvezes );
      
      Console.Read();

    } // Main().fim
  } // class.fim
} // namespace.fim



Dentro do namespace System tem a classe estática Environment que provê
muitas informações sobre o sistema. Entre essas informações estão os 
nome do usuário local e o nome de rede do computador recuperados pelos
atributos Environment.machineName e Environment.userName. Para encontrar
essa classe, no visualizador de objetos procure dentro de mscorlib >> 
System >> Environment.
