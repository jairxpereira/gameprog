03.2 Vertex Buffer
1.1 Visão geral






Em  nossas  aplicações de exemplo  até  o  presente  momento a função
montar_triangulo() é chamada todas as vezes pela função Renderizar(),
isto quer dizer que nossa forma geométrica é montada todas  as  vezes
antes da renderização. Essa remontagem constante consome tempo. Outro
aspecto negativo nesse processo é que existe  uma perda de tempo para
que  o  DirectX  transfira estes vértices para a memória da  placa de
vídeo.

Para  aumentar  a  eficência  da  renderização  pode-se  utilizar  um 
recipiente  especial  para  vértices  criado pela classe VertexBuffer. 
Com  este  recipiente  geralmente a forma geométrica é montada apenas 
uma vez e transferida para a placa de vídeo e remontada apenas quando
é necessária pela aplicação.

A reconfiguração do buffer de vértices é necessária após o  reset  do
dispositivo  que  pode  ser  ocasionado pela ação do usuário ou algum
evento  originado pelo  sistema  Windows; por exemplo, toda vez que o 
usuário  modifica  o tamanho da janela, o objeto VertexBuffer precisa 
ser recriado. Essa recriação do buffer de vértices geralmente é feita
no  método  que responde ao evento que notifica que o buffer deve ser
criado. 

A função  Renderizar()  ganha  uma modificação importante configurada 
pela  necessidade  de  informar  a origem dos vértices da forma a ser
renderizada. 

Formato de vértice
O buffer de vértices também é configurado para determinado formato de
vértice. Nossa aplicação vai utilizar  o posicionado e colorido  para
ter cor e as vantagens das transformações geométricas.

1.2 Estrutura da aplicação




O membro triangulo continua sendo um recipiente  de  vértices,  porém
agora não é o principal. Mas continua aqui pela necessidade  do passo
a passo no caminho de aprendizagem do complicado DirectX. Esse membro
vai receber 6 vértices de composição do quadrado.   Os triângulos vão
ser  montados  apena  uma vez em initGfx(), vão ser transferidos para
vbQuadrado  e  retransferidos  para o buffer de vértices toda vez que
ocorrer um evento que necessita essa correção.

O método quandoVertexBufferCriado() responde ao evento que notifica a
necessidade  de  criar  ou  recriar  o  buffer  e  é  responsável por 
retransfirir os triângulos para o VertexBuffer nas ocasiões em  que o
dispositivo sofrer um reset. 

A  função  Renderizar()  ganha mudanças importantes para acomodar  o
uso do VertexBuffer. Vamos ter a função device.SetStreamSource() que
aponta  para  o  buffer  de  vértices e  device.DrawPrimitives() que
renderiza as formas geométricas contidas nesse recipiente.

2.1 Definindo o recipiente do quadrado

Aqui definimos um VertexBuffer que será inicializado e preenchido com
os 6 vértices da array triangulo na função initGfx().

2.2 Inicialização gráfica: initGfx()

public void initGfx()
{

// Configuração dos parâmetros de apresentação
PresentParameters pps = new PresentParameters();
pps.Windowed = true;
pps.SwapEffect = SwapEffect.Discard;

// Adaptador default, processamento no hardware, processamento de vértice 
// no software, janela (this), parâmetros de apresentação (pps)
int adaptador = 0;
device = new Device(adaptador, DeviceType.Hardware, this,
CreateFlags.SoftwareVertexProcessing, pps);

// Montamos inicialmente nossos triângulos 
montar_triangulos();

// O buffer de vértices vbQuadrado é criado. 
vbQuadrado = new VertexBuffer(typeof(CustomVertex.PositionColored),
6, device, Usage.Dynamic | Usage.WriteOnly,
CustomVertex.PositionColored.Format,
Pool.Default);

// O buffer de vértices recebe os vértices da forma geométrica 
vbQuadrado.SetData(triangulo, 0, LockFlags.None);

// O buffer de vértices recebe a função quandoVertexBufferCriado()
// para responder ao evento onCreate() gerado pelo dispositivo 
vbQuadrado.Created += new EventHandler(this.quandoVertexBufferCriado);

} // initGfx()



// Montamos inicialmente nossos triângulos 
montar_triangulos();
A forma   geométrica   é  montada  apenas  uma  vez  no  processo  de 
inicialização do ambiente gráfico e somente  será   remontada  quando 
necessário na ocasião de um reset do dispositivo gráfico,  mas  ainda
assim  apenas  a  array  triangulo  será  repassada  para o buffer de 
vértices, vbQuadrado.

Inicialização do VertexBuffer
vbQuadrado = new VertexBuffer(typeof(CustomVertex.PositionColored),6, 
device, Usage.Dynamic | Usage.WriteOnly,
CustomVertex.PositionColored.Format, Pool.Default); 

Usage.Dynamic | Usage.WriteOnly
Estas  opções  configuram o gerenciamento de uso  do  VertexBuffer; o
primeiro flag determina que o buffer vai ser dinâmico que implica que
ele  vai  ser  alocado  na  memória  da  placa  de  vídeo;     o flag 
Usage.WriteOnly indica que o VertexBuffer  não  será  requerido  para
posterior leitura,  o que torna o seu processamento ainda mais rápido.

Pool.Default
Essa opção determina a classe da  área de memória  aonde  deve  ficar 
alocado  o  VertexBuffer  que  poderia  ser, entre outras opções,  em 
memória  gerenciável ou na  memória  de  vídeo. Pool.Default deixa  o 
DirectX  livre  para escolher o local  mais apropriado para acesso do
dispositivo e  ainda de acordo com as opções de gerenciamento de uso.


vbQuadrado.SetData(triangulo, 0, LockFlags.None); 
Aqui de fato o VertexBuffer recebe os vértices. As opções ao lado dos
vértices passados referem-se ao trancamento de alguma região de dados
do VertexBuffer. Aqui no caso nada está trancado com a placa de vídeo
apropriando-se totalmente dos dados passados.


vbQuadrado.Created += new EventHandler(this.quandoVertexBufferCriado);
Aqui é passado para o VertexBuffer a função que trata  o  evento  que
notifica quando ele é criado ou recriado;  quandoVertexBufferCriado()
assume  o trabalho de recompor o buffer de vértices  na ocorrência do
reset  do  dispositivo  ou  outro evento  que  danifique  os dados do
VertexBuffer.

2.3 Montando os triângulos
Esta função não sofreu alteração para acomodar-se ao VertexBuffer, mas
na nova dinâmica da aplicação ela é chamada apenas uma vez para montar
a forma geométrica a ser passada para o buffer de vértices. 


public void montar_triangulos()
{
// Cores para os vértices 
const int vermelho = 0xFF0000;
const int verde = 0x00FF00;
const int azul = 0x0000FF;

// É necessário 3 vértices para compor cada triângulo 
triangulo = new CustomVertex.PositionColored[6];

// Vetores para guardar a posição (x,y,z) dos vértices        
Vector3 p0, p1, p2, p3, p4, p5;

// Os vértices estão alinhados na mesma posição de profundidade 
float zpos = 1.0f;

// Define a cor dos vértices 
triangulo[0].Color = vermelho;
triangulo[1].Color = verde;
triangulo[2].Color = azul;

triangulo[3].Color = vermelho;
triangulo[4].Color = azul;
triangulo[5].Color = verde;

// Posições para os 3 vértices do primeiro triângulo 
p0 = new Vector3(-1.0f, 1.0f, zpos);
p1 = new Vector3(-1.0f, -1.0f, zpos);
p2 = new Vector3(1.0f, -1.0f, zpos);

// Posições para os 3 vértices do segundo triângulo
p3 = new Vector3(-1.0f, 1.0f, zpos);
p4 = new Vector3(1.0f, -1.0f, zpos);
p5 = new Vector3(1.0f, 1.0f, zpos);

// Posição dos vértices nos triângulos
triangulo[0].SetPosition(p0);
triangulo[1].SetPosition(p1);
triangulo[2].SetPosition(p2);

triangulo[3].SetPosition(p3);
triangulo[4].SetPosition(p4);
triangulo[5].SetPosition(p5);

} // montar_triangulos().fim 



2.4 Renderizando o quadrado

public void Renderizar()
{
// Limpa os dispositivos e os buffers de apoio 
device.Clear(ClearFlags.Target, cor_fundo, 1.0f, 0);

device.BeginScene();
// (...) Todo o código de desenhar vai aqui   

// Configura formato do vértice 
device.VertexFormat = CustomVertex.PositionColored.Format;

// Atualiza a câmera 
AtualizarCamera();

// O renderizador é informado da origem dos dados 
int nStream = 0;
int nDeslocamento = 0;
device.SetStreamSource(nStream , vbQuadrado, nDeslocamento);

// O quadrado é aqui renderizado 
device.DrawPrimitives(PrimitiveType.TriangleList, nDeslocamento, 2);

device.EndScene();

// Apresenta a cena renderizada na tela 
device.Present();

// Libera a janela para processar outros eventos 
Application.DoEvents();
} // Renderizar()



// O renderizador é informado da origem dos dados 
int nStream = 0;
int nDeslocamento = 0;
device.SetStreamSource(nStream , vbQuadrado, nDeslocamento);
Veja que o método SetStreamSource() identifica a  fonte  de  dados da
forma a ser renderizada. Por enquanto não se preocupe com streams  ou
deslocamentos  mas  saiba  que  a  partir  disso é possível empacotar
vários objetos 3d em uma mesma fonte de dados. E quando se tem vários
objetos  num  só  pacote  é necessário indicar o intervalo do que  se 
quer renderizar.

2.5 Recriando o buffer de vértices

// Remonta o buffer de vértice no evento onCreate 
private void quandoVertexBufferCriado(object sender, EventArgs e)
{
int nInicioDeslocamentoTrancado = 0;
VertexBuffer buffer = (VertexBuffer)sender;
buffer.SetData(triangulo, nInicioDeslocamentoTrancado, LockFlags.None);

} // quandoVertexBufferCriado().fim



int nInicioDeslocamentoTrancado = 0;
VertexBuffer buffer = (VertexBuffer) sender;
buffer.SetData(triangulo, nInicioDeslocamentoTrancado, LockFlags.None);
Esta função simplesmente recebe a referência do  objeto  VertexBuffer
que  deve  ser recomposto  com o conteúdo anterior.  Aqui no caso, os
vértices da array triangulo são repassados para  o buffer de vértices
sem preocupação em trancar alguma parte de seu conteúdo.


3. Código fonte do projeto de exemplo: prj_VertexBuffer




// prj_VertexBuffer - Arquivo: Tela.cs
// Esse projeto mostra como utilizar o recipiente de vértices
// produzido pela classe VertexBuffer.
// Produzido por www.gameprog.com.br
using System;
using System.Drawing;
using System.ComponentModel;
using System.Windows.Forms;
using Microsoft.DirectX;
using Microsoft.DirectX.Direct3D;

namespace prj_VertexBuffer
{
  public partial class Tela : Form
  {
    // Para criação do dispositivo gráfico
    private Device device = null;

    // Branco cor de fundo para nossa janela 
    private int cor_fundo = 0xFFFFFF;

    // Recipiente para os vértices dos triângulos
    // SetPosition() desse formato de vértice aceita uma
    // estrutura Vector3 com uma posição (x, y, z)
    private CustomVertex.PositionColored[] triangulo;

    // Memória ou buffer de vértices para o quadrado
    private VertexBuffer vbQuadrado = null;

    // Essa variável atualizada a cada ciclo ocasionará 
    // a animação do quadrado
    private float angulo = 0.0f;

    public Tela()
    {

      // Inicialização dos componentes.
      InitializeComponent();

      // Toda renderização deverá ocorrer dentro do evento onPaint()
      // Isso evita interferência estrangeira de processamento default
      // do sistema Windows
      this.SetStyle(ControlStyles.AllPaintingInWmPaint | ControlStyles.Opaque, true);

    } // construtor

    public void initGfx()
    {

      // Configuração dos parâmetros de apresentação
      PresentParameters pps = new PresentParameters();
      pps.Windowed = true;
      pps.SwapEffect = SwapEffect.Discard;

      // Adaptador default, processamento no hardware, processamento de vértice 
      // no software, janela (this), parâmetros de apresentação (pps)
      int adaptador = 0;
      device = new Device(adaptador, DeviceType.Hardware, this,
        CreateFlags.SoftwareVertexProcessing, pps);

      // Montamos inicialmente nossos triângulos 
      montar_triangulos();

      // O buffer de vértices vbQuadrado é criado.
      vbQuadrado = new VertexBuffer(typeof(CustomVertex.PositionColored),
         6, device, Usage.Dynamic | Usage.WriteOnly, 
         CustomVertex.PositionColored.Format,
         Pool.Default);

      // O buffer de vértices recebe os vértices da forma geométrica
      vbQuadrado.SetData(triangulo, 0, LockFlags.None);

      // O buffer de vértices recebe a função quandoVertexBufferCriado()
      // para responder ao evento onCreate() gerado pelo dispositivo
      vbQuadrado.Created += new EventHandler(this.quandoVertexBufferCriado);

    } // initGfx()

    private void AtualizarCamera()
    {
      // Dados para a configuração da matriz de projeção
      int largura = this.Width; // largura da janela
      int altura = this.Height;  // altura da janela      
      float aspecto = largura / altura; // aspecto dos gráficos      
      float campo_visao = (float)Math.PI / 4; // Campo de visão
      float corte_perto = 1.0f;
      float corte_longe = 100.0f;

      // Atualiza angulo para dar movivento
      angulo += 0.05f;

      // Mostra a parte interna do polígono
      // Experimente desativar essa linha com a instrução de comentário      
      device.RenderState.CullMode = Cull.None;

      // Utiliza iluminação default
      // Experimente desativar essa linha com a instrução de comentário  
      device.RenderState.Lighting = false;

      // Configura a matriz de projeção
      device.Transform.Projection = Matrix.PerspectiveFovLH(campo_visao,
          aspecto, corte_perto, corte_longe);

      // Rotaciona o triangulo indiretamente através da rotação dos
      // eixos da matriz mundial.
      device.Transform.World = Matrix.RotationZ(angulo);

      // Dados para a configuração da matriz de visualização
      Vector3 cam_pos = new Vector3(0, 0, 5.0f); // Posição da camera
      Vector3 cam_alvo = new Vector3(0, 0.0f, 0); // Alvo da câmera
      Vector3 cam_orientacao = new Vector3(0, 1.0f, 0); // Orientação da câmera          

      // Configura a matriz de visualização
      device.Transform.View = Matrix.LookAtLH(cam_pos, cam_alvo, cam_orientacao);
    }  // AtualizarCamera()

    public void Renderizar()
    {
      // Limpa os dispositivos e os buffers de apoio
      device.Clear(ClearFlags.Target, cor_fundo, 1.0f, 0);

      device.BeginScene();
      // (...) Todo o código de desenhar vai aqui   

      // Configura formato do vértice
      device.VertexFormat = CustomVertex.PositionColored.Format;
    
      // Atualiza a câmera
      AtualizarCamera();

      // O renderizador é informado da origem dos dados
      int nStream = 0;
      int nDeslocamento = 0;
      device.SetStreamSource(nStream , vbQuadrado, nDeslocamento);
      
      // O quadrado é aqui renderizado
      device.DrawPrimitives(PrimitiveType.TriangleList, nDeslocamento, 2);

      device.EndScene();

      // Apresenta a cena renderizada na tela
      device.Present();

      // Libera a janela para processar outros eventos
      Application.DoEvents();
    } // Renderizar()

    protected override void OnPaint(PaintEventArgs e)
    {
      // Trate outros processos padrões
      base.OnPaint(e);

      // Renderize a cena
      this.Renderizar();

      // Invalide para chamar novamente onPaint()
      this.Invalidate();
    } // onPaint().fim 


    public void montar_triangulos()
    {
      // Cores para os vértices
      const int vermelho = 0xFF0000;
      const int verde = 0x00FF00;
      const int azul = 0x0000FF;

      // É necessário 3 vértices para compor cada triângulo
      triangulo = new CustomVertex.PositionColored[6];

      // Vetores para guardar a posição (x,y,z) dos vértices        
      Vector3 p0, p1, p2, p3, p4, p5;

      // Os vértices estão alinhados na mesma posição de profundidade
      float zpos = 1.0f;

      // Define a cor dos vértices
      triangulo[0].Color = vermelho;
      triangulo[1].Color = verde;
      triangulo[2].Color = azul;

      triangulo[3].Color = vermelho;
      triangulo[4].Color = azul;
      triangulo[5].Color = verde;

      // Posições para os 3 vértices do triangulo
      p0 = new Vector3(-1.0f, 1.0f, zpos);
      p1 = new Vector3(-1.0f, -1.0f, zpos);
      p2 = new Vector3(1.0f, -1.0f, zpos);

      // Posições para os 3 vértices do segundo triangulo
      p3 = new Vector3(-1.0f, 1.0f, zpos);
      p4 = new Vector3(1.0f, -1.0f, zpos);
      p5 = new Vector3(1.0f, 1.0f, zpos);

      // Posição do primeiro vértice
      triangulo[0].SetPosition(p0);
      triangulo[1].SetPosition(p1);
      triangulo[2].SetPosition(p2);

      triangulo[3].SetPosition(p3);
      triangulo[4].SetPosition(p4);
      triangulo[5].SetPosition(p5);

    } // montar_triangulos().fim 

    // Remonta o buffer de vértice no evento onCreate
    private void quandoVertexBufferCriado(object sender, EventArgs e)
    {
      int nInicioDeslocamentoTrancado = 0;
      VertexBuffer buffer = (VertexBuffer)sender;
      buffer.SetData(triangulo, nInicioDeslocamentoTrancado, LockFlags.None);

    } // quandoVertexBufferCriado().fim


  } // fim da classe
} // fim do namespace
