1.2 A primeira janela

1.1 Visão geral





A experiência mais traumática para quem está começando a fazer jogos com o
DirectX é a inicialização do ambiente,  mostrar  a  primeira  janela  e  o 
primeiro triangulo renderizado.

Para mostrar uma janela turbinada com o motor do DirectX você vai proceder
desta forma:

1- Criar um projeto Windows Forms e configurar o ambiente geral da janela;

2- Inicializar o motor gráfico do DirectX;
   2.1 Configurar os parâmetros de apresentação a  ser  utilizado  em  um 
   dispositivo gráfico.

   2.2 Criar um dispositivo gráfico e  conectá-lo  com  a  janela  criada;

3- Configurar a câmera;

4- Renderizar a cena no evento onPaint(). 

O evento onPaint ocorre toda vez que sua janela  precisa  ser  atualizada.
Numa  aplicação  comum essa necessidade ocorre apenas quando a janela tem
seu  conteúdo  obstruido por outra janela e deve ser refeita na parte que
foi coberta. Numa  aplicação  DirectX  o  evento  onPaint()  vai  ocorrer 
continuamente.

1.1 Estrutura da aplicação

As classes dessa primeira  aplicação  estão  ilustradas  acima:  a  classe
Program  que  contém  a  função  Main() que chama a nossa janela produzida
pela  classe Tela responsável pelo gerenciamento da funcionalidade gráfica
da aplicação. 

Na  ilustração  acima  tem  algumas  funções  na  cor cinza que são criadas 
automaticamente pelo editor visual. Estas funções geralmente inicializam os
controles gráficos adicionados na janela principal e   no  encerramento  da
aplicação encarregam-se  de limpar o ambiente.   Estas  funções  não  serão 
discutidas a fundo como faremos com as outras nos tópicos seguintes.

2. Configurando o ambiente geral da janela

Crie um novo projeto Windows Forms Application e  configure a  propriedade  
name  da  janela principal como Tela.

No construtor da Janela,  depois da  chamada de InitializeComponent() você
deve acrescentar esta linha para que todo o  processo  de  desenho  ocorra
dentro do evento onPaint():

this.SetStyle(ControlStyles.AllPaintingInWmPaint | ControlStyles.Opaque, true);

O construtor da janela vai ficar com esta aparência:

public Tela()
{
 InitializeComponent();
 this.SetStyle(ControlStyles.AllPaintingInWmPaint | ControlStyles.Opaque, true);
}

Acima do construtor, na área de  declaração  de  métodos  e  propriedades,
declare  duas  variáveis  privadas  que  vamos  usar  posteriormente:

private Device device = null;     // Para criação do dispositivo gráfico
private int cor_fundo = 0xFFFFFF; // Branco cor de fundo para nossa janela


2.1 Inicialização do motor gráfico


public void initGfx()
{

// Configuração dos parâmetros de apresentação
PresentParameters pps = new PresentParameters();

// Aplicação vai rodar em modo janela
pps.Windowed = true; 

// Descarta o que não está pronto para ser renderizado
pps.SwapEffect = SwapEffect.Discard;

// Adaptador de vídeo default
int adaptador = 0;

// Monta o dispositivo gráfico utilizando a placa de vídeo default,
// renderização na placa de vídeo, tratamento de vértices
// pelo software(Directx), utilizando a janela (this), com
// as configurações de apresentação apontada (pps)
device= new Device(adaptador, DeviceType.Hardware, this,
CreateFlags.SoftwareVertexProcessing, pps);      

// Configuração da câmera
SetupCamera();      
} // initGfx() 



Geralmente você vai precisar de uma função para fazer isso. Vamos chamar
essa função de initGfx(). Nela vai haver estas linhas:

// Configuração dos parâmetros de apresentação 
PresentParameters pps = new PresentParameters();

Com esta linha acima é criada o objeto pps que vai conter as configurações
de apresentação da cena renderizada.

Na sequência configuramos a propriedade Windowed para verdadeiro para 
estabelecer que aplicação é uma janela em contraposição à tela cheia:
pps.Windowed = true; 

pps.SwapEffect = SwapEffect.Discard;
Esta linha acima informa que o dispositivo gráfico deve descartar
conteúdo do buffer gráfico que não estiver pronto para ser exibido.

Pronto, criamos e configuramos os parametros de apresentação! Agora
vamos criar o dispositivo gráfico:
 
 int adaptador = 0;
 device= new Device(adaptador, DeviceType.Hardware, this, 
        CreateFlags.SoftwareVertexProcessing, pps);    
		
Com estas linhas criamos o dispositivo gráfico. O adaptador configurado
como valor zero (0) aponta para a placa gráfica default do sistema;
o parâmetro DeviceType.Hardware informa que a renderização vai ser
feita pela placa gráfica. 

O parâmetro this aponta a janela como local aonde vai ser exibido a
cena.

CreateFlags.SoftwareVertexProcessing - informa que o processamento
dos vértices vai ser feito via Software, no caso, o DirectX.

pps representa a configuração dos parâmetros de apresentação.

Com isto criamos e configuramos o dispositivo gráfico. Dentro 
dessa função initGfx() vamos  chamar  setupCamera()  na  qual 
faremos a configuração inicial da camera.
	
2.2 Configurando a camera


private void SetupCamera()
{

// Dados para configurar a matriz de projeção
int largura = this.Width; // largura da janela
int altura = this.Height;  // altura da janela      
float aspecto = largura / altura; // aspecto dos gráficos      
float campo_visao = (float)Math.PI / 4; // Campo de visão
float corte_perto = 1.0f;
float corte_longe = 100.0f;

// Configura a matriz de projeção
device.Transform.Projection = Matrix.PerspectiveFovLH(campo_visao, 
        aspecto, corte_perto, corte_longe);

        
// Dados para configurar a matriz de visualização
Vector3 cam_pos = new Vector3(0, 0, 18.0f); // Posição da camera
Vector3 cam_alvo = new Vector3(0, 1.0f, 0); // Alvo da câmera
Vector3 cam_orientacao = new Vector3(0, 1.0f, 0); // Orientação da câmera

// Configura a matriz de visualização
device.Transform.View = Matrix.LookAtLH(cam_pos, cam_alvo, cam_orientacao);

} // SetupCamera().fim


A configuração da camera tem 3 aspectos para considerar:

- a posição da câmera 
- o alvo ou foco da camera 
- orientação da camera que define o que é cima ou baixo para a camera

Todos estes três aspectos são definidos em termos de (x,y,z). Para definir
estes três aspectos você deve usar uma estrutura Vector3():

Vector3 cam_pos = new Vector3(0, 0, 18.0f); // Posição da camera
Vector3 cam_alvo = new Vector3(0, 1.0f, 0); // Alvo da câmera
Vector3 cam_orientacao = new Vector3(0, 1.0f, 0); // Orientação da câmera
device.Transform.View = Matrix.LookAtLH(cam_pos, cam_alvo, cam_orientacao);

Conforme visto acima, a propriedade device.Transform.View do dispositivo
recebe  a  matriz  de  transformação  da  camera   através   da   função 
Matrix.LookAtLH().

Ainda antes de fazer esta etapa  da  configuração  da  câmera  você  vai
configurar  a  matriz  de  projeção da imagem que vai definir o campo de 
visão da camera, o aspecto da imagem que é uma relação da largura\altura
da tela e vai definir o espaço de enquadramento definindo o que é "perto"
ou "longe" da câmera:

int largura = this.Width; // largura da janela
int altura = this.Height;  // altura da janela      
float aspecto = largura / altura; // aspecto dos gráficos      
float campo_visao = (float)Math.PI / 4; // Campo de visão
float corte_perto = 1.0f;
float corte_longe = 100.0f;     
device.Transform.Projection = Matrix.PerspectiveFovLH(campo_visao, 
		aspecto, corte_perto, corte_longe);
		
Conforme  visto  acima,  a  propriedade  device.Transform.Projection  do
dispositivo  recebe  a  matriz  de  projeção da camera através da função 
Matrix.PerspectiveFovLH().

2.3 Renderizando a cena


public void Renderizar()
{
  // Limpa o dispositivo
  device.Clear(ClearFlags.Target, cor_fundo, 1.0f, 0);

  // inicia o processo de renderização
  device.BeginScene();

  // (...) Todo o código de desenhar vai aqui     
 
 // Finaliza o processo de renderização
 device.EndScene();

  // Apresenta a cena renderizada nos bastidores
  device.Present();
  
  // Verifica outros eventos para não travar a aplicação
  Application.DoEvents();
} // Renderizar().fim


Vamos utilizar uma função que vamos chamar de Renderizar() que vai ter este
conteúdo:

device.Clear(ClearFlags.Target, cor_fundo, 1.0f, 0); 
Esta função limpa o dispositivo e os buffers utilizados.  Ela limpa a tela
e configura a cor de fundo do dispositivo, branco em nosso exemplo.

ClearFlags.Target - limpa a superfíce de renderização, no caso nossa
janela.

cor_fundo - É o branco que foi definido anteriormente.

Os valores 1.0f e 0 não precisam ser discutidos agora. Configure
sempre estes valores. Eles se referem a buffers de profundidade
que serão discutidos adiante em nosso curso.

device.BeginScene();           
// Todo o código de desenhar vai aqui neste miolo.
device.EndScene();  

Todo o processo de desenhamento da cena ocorre entre device.BeginScene()
e device.EndScene() que monta e desmonta o ambiente final de renderização.
É nesse miolo que vamos colocar todas as nossas funções que desenham
nossos objetos.


device.Present();
Esta linha apresenta a cena na tela. Como não desenhamos nada, apenas a 
janela em branco vai aparecer.


Application.DoEvents(); 
Esta linha libera nossa aplicação para processar outros eventos do
Windows sobre nossa janela. Por exemplo, o Windows vai processar 
os comandos minimizar\maximizar que o usuário realizar sobre nossa
janela. 

2.4 O evento onPaint()


protected override void OnPaint(PaintEventArgs e)
{
  base.OnPaint(e);
  this.Renderizar();
  this.Invalidate();
} // onPaint().fim 


Definimos aqui uma função para processar o evento onPaint. Essa função
vai chamar continuamente nossa função Renderizar().

base.OnPaint(e);
Pede para o Windows processar processos padrões

this.Renderizar();
Chama nossa renderização

this.Invalidate(); 
Invalida a tela para chamar novamente OnPaint()!

3. Rodando nossa aplicação e mostrando nossa janela

Na classe Program, na função Main(), apague o código gerado automaticamente
e substitua por estas linhas: 

using (Tela tela = new Tela())
{
 tela.Show();
 tela.initGfx();
 Application.Run(tela);      
} // end using 	  
	  
Isso garante que nossa função initGfx() vai ser chamada na ordem correta e
Application.Run()  realiza  processos internos  como encerrar corretamente
a aplicação. A construção using {}  garante  que  os  recursos  utilizados
sejam liberados no final do bloco.

    static class Program

        static void Main()

                Application.Run(tela);

        } // Main().fim
    } // fim da classe
} // fim do namespace





// Prj_Janela - Arquivo: Janela.cs
// Esse projeto ilustra como inicializar o motor gráfico do DirectX 9
// Produzido por www.gameprog.com.br
using System;
using System.Drawing;
using System.ComponentModel;
using System.Windows.Forms;
using Microsoft.DirectX;
using Microsoft.DirectX.Direct3D;

namespace prj_Janela
{
    public partial class Tela : Form
    {
        private Device device = null;     // Para criação do dispositivo gráfico
        private int cor_fundo = 0xFFFFFF; // Branco cor de fundo para nossa janela   

        public Tela()
        {
            // Qualquer configuração em algum componente, faça depois dessa função!
            // Pois a geração automática de código pode suprimir algo que você
            // fizer antes dela
            InitializeComponent();

            // Toda renderização deverá ocorrer dentro do evento onPaint()
            // Isso evita interferência estrangeira de processamento default
            // do sistema Windows
            this.SetStyle(ControlStyles.AllPaintingInWmPaint |
		 ControlStyles.Opaque, true);
        } // construtor

        public void initGfx()
        {

            // Configuração dos parâmetros de apresentação
            PresentParameters pps = new PresentParameters();
            pps.Windowed = true;
            pps.SwapEffect = SwapEffect.Discard;

            // Adaptador default, processamento no hardware, processamento de vértice 
            // no software, janela (this), parâmetros de apresentação (pps)
            int adaptador = 0;
            device = new Device(adaptador, DeviceType.Hardware, this,
              CreateFlags.SoftwareVertexProcessing, pps);

            // Configura a camera
            SetupCamera();

        } // initGfx()

        private void SetupCamera()
        {

            // Dados para a configuração da matriz de projeção
            int largura = this.Width; // largura da janela
            int altura = this.Height;  // altura da janela      
            float aspecto = largura / altura; // aspecto dos gráficos      
            float campo_visao = (float)Math.PI / 4; // Campo de visão
            float corte_perto = 1.0f;
            float corte_longe = 100.0f;

            // Configura a matriz de projeção
            device.Transform.Projection = Matrix.PerspectiveFovLH(campo_visao,
                aspecto, corte_perto, corte_longe);

            // Dados para a configuração da matriz de visualização
            Vector3 cam_pos = new Vector3(0, 0, 18.0f); // Posição da camera
            Vector3 cam_alvo = new Vector3(0, 1.0f, 0); // Alvo da câmera
            Vector3 cam_orientacao = new Vector3(0, 1.0f, 0); // Orientação da câmera

            // Configura a matriz de visualização
            device.Transform.View = Matrix.LookAtLH(cam_pos, cam_alvo, cam_orientacao);
        }  // setupCamera()

        public void Renderizar()
        {
            // Limpa os dispositivos e os buffers de apoio
            device.Clear(ClearFlags.Target, cor_fundo, 1.0f, 0);

            device.BeginScene();
            // (...) Todo o código de desenhar vai aqui   
            device.EndScene();

            // Apresenta a cena renderizada na tela
            device.Present();

            // Libera a janela para processar outros eventos
            Application.DoEvents();
        } // render()

        protected override void OnPaint(PaintEventArgs e)
        {
            // Trate outros processos padrões
            base.OnPaint(e);

            // Renderize a cena
            this.Renderizar();

            // Invalide para chamar novamente onPaint()
            this.Invalidate();
        } // onPaint().fim 

    } // fim da classe
} // fim do namespace
