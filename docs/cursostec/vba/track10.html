 10. Subrotina e o comando GOSUB/RETURN 


        No capítulo sobre rótulo você aprendeu que você pode dar nomes
        para certos trechos de seu programa, e que pode usar este nome
        como referência para saltos condicionais e incondicionais.

        Estes trechos nomeados ou rotulados de seus programas são chamados
        de subrotinas. A subrotina executa uma pequena tarefa que contribui
        para a obtenção do objetivo total do programa.

        GOSUB NomeDaSubRotina
        código
        END

        NomeDaSubRotina:
        código
        RETURN

        O comando GOSUB desvia a execução do programa para a subrotina
        especificada. O código da subrotina é executado e assim que o
        Qbasic encontra o comando RETURN, o programa retorna ao ponto
        no qual a subrotina havia sido chamada. Depois do  o VBA
        executa as linhas seguintes imediamente abaixo do comando GOSUB.

        Geralmente, as subrotinas contém código que vai ser rodado várias
        vezes durante a execução programa.

        Exemplo:

		Dim x, y As Integer
		Dim txt As String

		inicio: 
		MsgBox "Multiplicando dois números, x por y." & Chr$(13) & _
		"Digite zero para encerrar"

		txt = InputBox("Digite o valor de x: ", "Coletando informações")
		x = Val(txt)
		If x = 0 Then GoTo final

		txt = InputBox("Digite o valor de y: ", "Coletando informações")
		y = Val(txt)
		If y = 0 Then GoTo final

		GoSub multiplica
		GoTo inicio

		final: 
		MsgBox "Fim de Programa"
		End

		multiplica: 
		mult% = x * y
		MsgBox Str$(x) & " vezes " & y & " é igual a " & Str$(mult)
		Return
