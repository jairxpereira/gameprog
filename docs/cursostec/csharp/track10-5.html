10-5 Listas: Passando pela coleção de objetos
1.1 Visão geral
Em nosso primeiro projeto com List<T> utilizamos uma instrução foreach para
acessar os elementos da nossa coleção de objetos.  Existe ainda duas outras
formas  de  passar  pelos  elementos  da  coleção utilizando o indexer ou o
iterador da classe.

1.2 O indexer: []
Adicionalmente é possível acessar os elementos usando o indexer cuja sintaxe
é a mesma  do acesso de elementos  de  uma  array  utilizando  os  colchetes 
através  dos  quais é obtido uma referência ao objeto em dada posição dentro
da coleção.  O  indexer  é  uma  funcionalidade manualmente adicionada a uma 
classe  semelhante  ao processo  de sobrecarregamento de operador no qual os
colchetes ganham este significado particular dentro da classe.

1.3 O iterador
Na construção de um laço for cada ciclo do laço é chamado de iteração. Esse
processo permite passar, por exemplo, pelos elementos de uma array de dados
do tipo inteiro. Rusticamente, neste exemplo,  o for poderia ser chamado de
o iterador do processo porque permite a realização da iteração.

Esse conceito foi estendido para um conjunto de funções que permite realizar
o processo de iteração e obter uma referência a um objeto. Em C#, o iterador
é  um  objeto  de  uma classe interna à classe principal que oferece funções
e  referências  para  acessar  os  objetos  contidos  na coleção.  O próprio 
funcionamento  da instrução foreach para ser usada com objetos depende que a
classe tenha implementado certos métodos com funcionalidades específicas  do
processo de iteração.
 
2.1 Passando pela coleção de objetos via indexer



int ntam;
ntam = lista_monstro.Count;

// Exibindo os monstros usando o indexer
Console.WriteLine(" \n\n Exibindo monstros usando o indexer");

for (int ncx = 0; ncx < ntam; ncx++)
lista_monstro[ncx].mostrar();


O indexer retorna uma referência a um objeto que pode ser guardada numa
variável  do tipo do objeto ou pode-se concatenar acesso a propriedades,
atributos ou métodos conforme fizemos no exemplo acima.


2.2 Passando pela coleção de objetos via iterador


// Exibindo monstros usando um iterador
Console.WriteLine(" \n\n Exibindo monstros usando um iterador");
List<Monstro>.Enumerator iter = new List<Monstro>.Enumerator();
iter = lista_monstro.GetEnumerator();

while (iter.MoveNext())
 {
  Monstro monster = iter.Current;
  monster.mostrar();
 } // endwhile
 

 
List<Monstro>.Enumerator iter = new List<Monstro>.Enumerator();
Com esta sintaxe definimos um iterador, que aqui denominamos iter, que é uma
estrutura com alguns métodos dentro da classe List.

iter = lista_monstro.GetEnumerator();
Aqui inicializamos o iterador  que agora aponta para um posição (-1) antes
do  início  dos  elementos.  O método MoveNext() posiciona o acesso para o
próximo elemento. A propriedade Current retorna uma referência ao elemento
corrente.

Esta não é uma sintaxe simpática para acessar os elementos  de  uma  coleção, 
na verdade estes métodos representam a implementação da interface IEnumerator
para que os elementos possam ser acessados através da instrução  foreach que
usa estes métodos nos bastidores do seu processo de iteração.  Compreende-se
aqui que estes são os métodos e a interface que deve ser implementada quando
a intenção  é  abrir  acesso  aos elementos de uma classe-coleção através da 
instrução foreach.

3. Código fonte do programa exemplo




// Projeto prj_List02
// Este programa mostra o uso de List<T> 
// Mostra o indexer e o iterador
using System;
using System.Collections.Generic;

namespace prj_List02
{
  class Monstro 
  {
    // Nome do monstro
    public string m_nome;
    // Capacidade de fazer estrago   
    public int m_estrago;

    // Construtor
    public Monstro(string nome, int estrago)
    {
      m_nome = nome;
      m_estrago = estrago;
    } // fim do construtor

    // Exibe o nome do monstro e capacidade de fazer estrago
    public void mostrar()
    {
      Console.WriteLine(" Mostro: ({0},{1})", m_nome, m_estrago);
    } // mostrar().fim
   
  } // fim da classe Monstro  


  class Program
  {
    static void Main(string[] args)
    {

      config_janela("prj_List02");    

      // Declarando uma lista de monstros
      List<Monstro> lista_monstro = new List<Monstro>();
     
      // Adicionando monstro na lista_monstro
      Monstro lobisomem = new Monstro("Lycan", 33);
      lista_monstro.Add(new Monstro("Godzila", 10));
      lista_monstro.Add(new Monstro("Dracula", 22));
      lista_monstro.Add(new Monstro("Shrek", 120));
      lista_monstro.Add(lobisomem);

      int ntam;
      ntam = lista_monstro.Count;
      Console.WriteLine("\n");

      // Exibindo os monstros usando o indexer
      Console.WriteLine(" \n\n Exibindo monstros usando o indexer");
      for (int ncx = 0; ncx < ntam; ncx++)
        lista_monstro[ncx].mostrar();

      // Exibindo monstros usando um iterador
      Console.WriteLine(" \n\n Exibindo monstros usando um iterador");
      List<Monstro>.Enumerator iter = new List<Monstro>.Enumerator();
      iter = lista_monstro.GetEnumerator();

      while (iter.MoveNext())
      {
        Monstro monster = iter.Current;
        monster.mostrar();
      }      

      // Pausa para a leitura da tela
      Console.ReadLine();
    } // main() fim

    // Método para configurar a janela
    private static void config_janela(string titulo)
    {
      // Vamos configurar a janela
      Console.ForegroundColor = ConsoleColor.Blue;
      Console.BackgroundColor = ConsoleColor.White;
      Console.Title = titulo;
      Console.SetBufferSize(80, 30);
      Console.SetWindowSize(80, 30);
      Console.Clear();
    } // config_janela() fim    

  } // fim da classe Program
} // fim do namespace prj_List02
