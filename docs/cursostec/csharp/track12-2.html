12-2 Criando uma classe-exceção particular
1.1 Visão geral
- A classe-exceção particular deve ser derivada da classe System.Exception
que vai lhe doar toda  a  funcionalidade necessária para ser utilizada com
throw, try e catch.

- O nome da classe deve conter o sufixo -Exception

1.2 Contexto da aplicação de exemplo
A aplicação  de  exemplo construirá a exceção HackeadoException que será 
sinalizada quando o programa perceber que suas variáveis foram alteradas
para além dos limites determinados pela aplicação. 

2. Elaborando uma classe-exceção


  class HackeadoException:System.Exception
  {
    public HackeadoException() : base("Os campos do objeto foram hackeados!") 
    { } // fim do primeiro construtor

    public HackeadoException(string mensagem): base(mensagem)
    { } // fim do segundo construtor
  } // fim da classe


Conforme a composição dos construtores, o objeto dessa classe poderá ser
criado dessas duas formas:


	
throw new HackeadoException();
throw new HackeadoException("Um item foi hackeado");



No primeiro caso, a string Message da classe base foi configurada com
uma mensagem padrão; o segundo  construtor  aceita  uma  string  como 
argumento que é repassada para a classe base.

3. Utilizando uma classe-exceção genérica
Mais lucrativo  do que  criar uma classe-exceção particular é utilizar
uma  classe  exceção  genérica  configurando apenas a mensagem emitida 
para  adequar-se  ao contexto da sua aplicação.  A criação de classes-
-exceções  tornam-se  um  pouco  mais complexas quando a aplicação vai
realizar operações  mais  sofisticadas  como  troca  remota  de  dados
necessitando um outro construtor mais especializado para este contexto.

Ademais,  cada  namespace  da  biblioteca .net  fornece  suas classes 
exceções contextuais que aborda melhor os problemas que podem ocorrer
na aplicação.  O  uso de exceções  para  tratamento de erros não deve
substituir a correção de bugs (erros de lógica, etc)  e  processos de
validação para garantir que as entradas de dados feitas pelo  usuário
estejam seguras para processamento posterior.

Exemplo de uso de uma classe exceção nativa da biblioteca .net:


// Melhor do que criar uma exceção é usar uma genérica
try
{

 if (nvidas > 9 || energia > 99) 
    throw new ApplicationException ("\n Configuração ruim!");
} // endtry

catch (ApplicationException erro)
{
 Console.WriteLine(" {0}", erro.Message);
 nvidas = 7;
 energia = 10;
 Console.WriteLine(" Configuração restaurada para valores padrões.");
} // endcatch
// **********************************************************************



4. Código fonte do projeto exemplo: prj_Exception02




// Projeto prj_Exception02 - Arquivo:Program.cs
// Este programa mostra a definição de uma exceção particular
using System;

namespace prj_Exception02
{
  class Program
  {
    static void Main(string[] args)
    {

      config_janela("prj_Exception02");

      // Variáveis de trabalho
      int energia = 100;
      int nvidas = 10;

      // Verifica se energia foi hackeada      
      try
      {
        int nval = energia;
        if (nval > 99)
 throw new HackeadoException();        
      } // endtry

      catch(HackeadoException erro)
      {
        Console.WriteLine(" {0}", erro.Message);
        Console.WriteLine(" Jogador foi excluído da partida.");
      } // endcatch
      // **********************************************************************

      
      // Verifica se nvidas foi hackeada
      try
      {
        int nval = nvidas;
        if (nval > 9) throw new HackeadoException("O valor nvidas foi hackeado!");
      } // endtry

      catch (HackeadoException erro)
      {
        Console.WriteLine(" {0}", erro.Message);
        nvidas = 1;
        Console.WriteLine(" O valor nvidas foi restaurado para o menor valor!");
      } // endcatch
      // **********************************************************************

      
      // Melhor do que criar uma exceção é usar uma genérica
      try
      {
        
        if (nvidas > 9 || energia > 99)
		 throw new ApplicationException ("\n Configuração ruim!");
      } // endtry

      catch (ApplicationException erro)
      {
        Console.WriteLine(" {0}", erro.Message);
        nvidas = 7;
        energia = 10;
        Console.WriteLine(" Configuração restaurada para valores padrões.");
      } // endcatch
      // **********************************************************************

      Console.Read();

    } // main() fim

    // Método para configurar a janel
    private static void config_janela(string titulo)
    {
      // Vamos configurar a janela
      Console.ForegroundColor = ConsoleColor.Blue;
      Console.BackgroundColor = ConsoleColor.White;
      Console.Title = titulo;
      Console.SetBufferSize(80, 30);
      Console.SetWindowSize(80, 30);
      Console.Clear();
      Console.WriteLine("");
    } // config_janela() fim    

  } // fim da classe Program
} // fim do namespace prj_Exception02
