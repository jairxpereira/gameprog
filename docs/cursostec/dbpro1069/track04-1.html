04-1. Desvio de fluxo de execução


 Você pode identificar partes do seu programa, essa identificação é chamada
 de label ou rótulo. O nome do rótulo é seguido pelos dois pontos (:).

 Por exemplo: 
                inicio:
                        código
                meio:
                        código
                fim:
                        código 

 A utilidade do rótulo é servir de referência para as instruções de desvio 
 de fluxo de programa, ocasionando saltos para qualquer parte do programa.

 goto

 goto rótulo

 O comando goto desvia a execução do programa para o rótulo especificado.

 Ex.: 
 goto final


 gosub...return

 gosub rótulo

 Os trechos nomeados do seu programa são denominados  rotinas  ou  subrotinas. 
 Você pode definir subrotinas cujo código geralmente é requisitado e executado
 diversas vezes ao longo da  execução  do  programa.  Depois  da  execução  da 
 subrotina, o fluxo retorna ao ponto  imediatamente  posterior  a  chamada  da 
 subrotina. Isso é feito usando  a dupla de comandos gosub...return.  O  código
 da subrotina é identificado com um rótulo e deve ser finalizado pela instrução
 return que força o retorno ao ponto de chamada da subrotina.

 Ex.: 

 gosub carrega_som

 ...

 carrega_som:

        código

 return 

Veja o programa exemplo:


 




Rem Project: rotinas
Rem Created: 02/08/2013 01:00:31

inicio:
' Configurando a janela
branco as integer = 0xFFFFFF
azul   as integer = 0x0000FF
set window on
set window title "rotinas.dba"
ink azul, branco
cls branco

print
print " ....inicio do programa"
gosub carrega_som
gosub carrega_imagem
gosub carrega_modelos

meio:
print " ....meio do programa"

goto final
print " Esse linha nunca é executada..."

carrega_imagem:
print " ....as imagens foram carregadas ok"
return

carrega_modelos:
print " ....os modelos foram carregados ok"
return

carrega_som:
print " ....os efeitos sonoros foram carregados ok"
return

final:
print
Print " [FIM] Pressione qualquer tecla para continuar"
suspend for key
end
